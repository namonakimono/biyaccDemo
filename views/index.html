Abstract
Language designers usually need to implement parsers and pretty- printers. Despite being two intimately related programs, they are de- signed and implemented separately. This approach has an obvious dis- advantage: as a language evolves, both its parser and pretty-printer need to be separately revised and kept synchronised. Such tasks are routine but complicated and error-prone. To facilitate these tasks, we propose a language called BiYacc, whose programs denote both a parser and a pretty-printer. In essence, BiYacc is a domain-specific language for writing putback-based bidirectional transformations — the pretty-printer is a putback transformation, and the parser is the corre- sponding get transformation. The pairs of parsers and pretty-printers generated by BiYacc are thus always guaranteed to satisfy the round- trip properties. Unconventionally, the pretty-printer — being a put- back transformation — accepts not only an abstract syntax tree but also a string, and produces an updated string consistent with the given abstract syntax tree. We can thus make use of the additional input string, with mechanisms such as simultaneous pattern matching on the view and the source and prioritising transformation rules, to describe more varied pretty-printing strategies.

A few notes:
  • The forward transforamtion may take much time, since it must try all the cases.
    We set the time limit for one transformation to 60 secondes.
    If run time exceeds 60 seconds, it will show "tranformation failed".
  • Please do not edit the abstract syntax and the concrete syntax. As stated in the paper, we have not implemented the functionality that automatically generated the parser and pretty-printer for the user input string and AST. So currently these tools are written by hand and binded to the syntax.

Installation Guide
Generally, you should install Haskell Platform, BiFluX, BiYacc in order. The detail of how to install them can be found in the "README" under root folder of the project.
After that, you can use our system by comand line. Or alternatively, installing the web-gui and testing using it. Installation of Web-GUI is very simple, you just need to follow the install.txt.

How to use command line ?
Three Steps:
  • Compile source to Haskell code. Typing "BiFluX --help" to see all the options.
  • Complie Haskell source to an executable file. Typing "ghc xx.hs" to compile it. xx.hs is the generated Haskell file form the previous step.
  • Execute bidirectionally. Typing "./xx --help" to see how to execute. XX is the generated executable file from the previous step.

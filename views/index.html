Abstract
Language designers usually need to implement parsers and pretty-printers. Despite being two intimately related programs, in practice they are designed and implemented separately. This approach has an obvious disadvantage: as a language evolves, both its parser and pretty-printer need to be separately revised and kept synchronised. Such tasks are routine but complicated and error-prone. To facilitate these tasks, we propose a language called BiYacc, whose programs denote both a parser and a pretty-printer. In essence, BiYacc is a domain-specific language for writing putback-based bidirectional transformations — the pretty-printer is a putback transformation, and the parser is the corresponding get transformation. The pairs of parsers and pretty-printers generated by BiYacc are thus always guaranteed to satisfy the usual round-trip properties. Unconventionally, the pretty-printer — being a putback transformation — accepts not only an abstract syntax tree but also a string, and produces an updated string consistent with the given abstract syntax tree. We can thus make use of the additional input string, with mechanisms such as simultaneous pattern matching on the view and the source and prioritising transformation rules, to describe more varied pretty-printing strategies.
Test Cases

We have tested our system with non-trivial examples. Please have fun with it.
A few notes:

  • The forward transformation may take much time, since it must try all the cases.
    We set the time limit for one transformation to 60 seconds.
    If run time exceeds 60 seconds, it will show "transformation failed".
  • As stated in the paper, we have not implemented the functionality that automatically generated the parser and pretty-printer for the user input string and AST. So currently these tools are written by hand and binded to the syntax. Thus the syntax is locked in this page.

Installation Guide
Generally, you should install GHC (The Glasgow Haskell Compiler), BiFluX, BiYacc in order.
As for GHC, please refer to https://www.haskell.org/ghc/.
As for BiFluX and BiYacc, please refer to the "README" file under the root folder of each project respectively.
After that, you can use our system at command line.

Alternatively, you can install this web-GUI in your own system.
Get the source firstly. Then please refer to the "install.txt" in the root folder of the web-GUI.
